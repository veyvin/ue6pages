This module is responsible for initializing the hardware audio device, managing output buffers, and ensuring that the high-level DSP (Digital Signal Processing) generated by systems like MetaSounds is converted into the PCM data that Apple hardware expects.

Handle iOS Background Audio
On iOS, the OS will “eliminate” your audio stream if the app enters the background unless configured otherwise. In Project Settings > Platforms > iOS, ensure the “Audio” Background Mode is checked if you need your game to continue playing sound during interruptions or while the device is locked.
Adjust Platform Headroom
To prevent digital clipping on mobile speakers, use the PlatformHeadroomDB setting in your IOS/MacEngine.ini. Setting this to a value like -3.0 or -6.0 provides a buffer for loud “elimination” sounds and high-intensity combat, ensuring the Core Audio limiters don’t distort your mix.

Manage Buffer Sizes for Latency
For rhythm-based games on Mac or iOS, you can tune the CallbackBufferFrameSize in your platform’s Engine settings. Smaller buffer sizes “eliminate” latency but increase the risk of CPU hitches causing “pops” or “clicks.” 512 frames is a standard balance, while 256 is ideal for low-latency feedback.
Respond to Audio Route Changes
Core Audio frequently sends notifications when a user connects AirPods or unplugs a wired headset. The AudioMixerCoreAudio module handles the device swap, but you should listen for FCoreDelegates::OnAudioRouteChanged to pause your game or update UI if the primary output device is “eliminated” during play.
Optimize for Apple Silicon
When building for Mac (M1/M2/M3), ensure you are using the modern Audio Mixer (enabled by default). The AudioMixerCoreAudio module is optimized for the ARM-based NEON instruction sets, allowing for more simultaneous MetaSound voices with less battery drain than legacy backends.
Use the Audio Bus for Side-Chaining
Because Core Audio is highly efficient with multi-channel routing, use the Unreal Audio Bus system to send signals between submixes. This is the best way to implement “ducking” (e.g., lowering music volume when an elimination bark occurs) while maintaining the high dynamic range supported by Apple hardware.
Debug via “stat audio”
If you experience stuttering on Apple devices, use the console command stat audio. This allows you to monitor the “Audio Render Thread” time. If the render time approaches the limit of your buffer size (e.g., 10ms for a 48kHz stream), you must “eliminate” expensive DSP effects or reduce the max voice count to prevent audio dropouts.
