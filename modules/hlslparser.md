### AI Description & Usage Tips

signed to tokenize, parse, and analyze High-Level Shading Language (HLSL) source code. It transforms raw shader text into an Abstract Syntax Tree (AST), which is a structured representation of the shader’s logic, variables, and functions.

This module is primarily used by the Material Translator and Niagara, as well as the CrossCompiler (HLSLCC). By providing a programmatic way to “understand” shader code, it allows the engine to validate syntax, perform optimizations, and convert HLSL into other shading languages (like GLSL or Metal) to ensure cross-platform compatibility. It helps eliminate the need for manual text-parsing when the engine needs to modify or inspect shader logic.

Practical Usage Tips and Best Practices
Validate Custom HLSL in Material Nodes
If you are writing a custom editor tool that generates shader code, use the HLSLParser to validate the syntax before attempting to compile. This allows you to catch and eliminate syntax errors early, providing the user with specific line numbers and error messages instead of a generic “Shader failed to compile” warning.
Analyze Shader Complexity
You can use the AST generated by this module to perform static analysis on shaders. By traversing the tree, you can count the number of instruction-heavy operations (like texture samples or complex loops) to eliminate unoptimized shaders from your project before they impact performance.
Perform Automated Refactoring
When upgrading your project to a newer engine version that deprecates certain shader functions, the HLSLParser can help you automate the search-and-replace process. Because it understands the code structure, it can eliminate the risk of accidental replacements that often occur with standard “find-and-replace” text tools.
Optimize Niagara Custom Modules
Niagara uses HLSL under the hood. For advanced VFX pipelines, you can use the parser to inspect custom HLSL scripts within Niagara to ensure that variables are being reused efficiently. This helps eliminate register pressure and improves GPU performance for complex particle systems.
Bridge Custom Shading Languages
If your studio uses a proprietary shading language, you can use the HLSLParser as a reference to build a translator that converts your custom code into Unreal-compatible HLSL. This helps eliminate the technical debt of manually rewriting shaders for Unreal Engine.
Debug Cross-Platform Translation Issues
If a shader works on PC but fails on mobile or console, the issue often lies in the cross-compilation stage. Use the parser to inspect the AST and see how the code is being interpreted before it is sent to the cross-compiler. This helps you eliminate platform-specific bugs by identifying incompatible HLSL structures.
Include in Editor-Only Dependencies
To use this module in C++, add "HLSLParser" to your PrivateDependencyModuleNames in your *.Build.cs file. Ensure this is wrapped in an editor-only module or protected by #if WITH_EDITOR to eliminate build failures when packaging the final shipping executable, as the parser is not needed at runtime.
Leverage for Shader “Minification”
By analyzing the AST, you can identify and eliminate dead code or unused variables in large .usf or .ush files. While the platform compilers usually do this, doing it at the HLSL level can lead to cleaner, more readable intermediate files during the debugging process.
