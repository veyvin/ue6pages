Practical Usage Tips and Best Practices
Optimize Iterative Cooking Performance
The CookMetadata system tracks which assets have changed since the last build. To “eliminate” long wait times for your team, ensure that the metadata cache is preserved on your build machines. This allows the cooker to skip unchanged assets, significantly reducing the “Cook-On-The-Fly” or staging time.
Handle Metadata Bloat in CI/CD
In large-scale projects, the cumulative metadata for thousands of assets can become massive. Periodically “eliminate” stale metadata by performing a “Clean Cook.” This ensures that the metadata manifest doesn’t contain references to assets that have been deleted from the source content folder.
Identify Heavy Dependencies
Use the logs generated by the CookMetadata module to identify assets that trigger massive “cascading” re-cooks. If a single material change causes the “elimination” of the cache for half the project, it indicates a dependency bottleneck that should be addressed in the asset architecture.
Strict Editor/Developer Boundary
This module is classified under Developer modules. Always ensure your module dependency in Build.cs is wrapped in a check for non-shipping builds. This helps “eliminate” unnecessary code and potential security risks from being compiled into the final shipping executable.
Validate Metadata for DLC and Patches
When creating DLC, the CookMetadata module is vital for ensuring that the new content correctly references the base game assets. Use the generated metadata to verify that you aren’t accidentally including base-game assets in your DLC package, which helps “eliminate” excessive patch sizes.
Leverage for Zen Store Integration
If your project uses Zen Store (the modern UE5 data delivery system), the CookMetadata module is instrumental in mapping assets to their Zen IDs. Ensure your Zen Server is correctly configured to store this metadata to “eliminate” I/O bottlenecks during asset loading in the editor and runtime.
Debug with stat Cooker
To see how much time the metadata generation and dependency tracking is taking, use the stat Cooker console command during a local cook. If metadata processing is a primary bottleneck, you may need to “eliminate” complex circular references in your Blueprints or Materials.
Utilize for Custom Asset Auditing
You can hook into the cooking process to add custom metadata fields to your assets. This is useful for “eliminating” unoptimized content from a build; for example, you can flag assets that exceed a certain memory threshold and have the cooker throw a warning or error based on the metadata.
