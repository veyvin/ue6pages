the FreeType open-source library. It is the engine’s primary low-level font engine, responsible for parsing, decoding, and rasterizing OpenType (OTF) and TrueType (TTF) font files into bitmaps or glyph data that Slate and UMG can render.

While most developers interact with fonts through Font Assets and Font Face Assets, the FreeType2 module works behind the scenes to handle font metrics, kerning, and the generation of the glyph atlases used by the engine’s Font Cache.

Practical Usage Tips and Best Practices
1. Choose the Right Loading Policy

In the Font Face Asset, you can choose how FreeType2 loads data.

Best Practice: Use Inline or Lazy Load for most UI fonts. Use Stream only for massive localized fonts with thousands of glyphs (like CJK fonts) to eliminate high initial memory overhead, though this may cause minor hitches during the first time a character is rendered.
2. Understand Hinting Modes

FreeType2 uses “Hinting” to align font outlines with the pixel grid for better readability at small sizes.

Tip: If your font looks “blurry” or “thin” at small sizes, try changing the Hinting mode to Monochrome or Auto Light. This adjusts how FreeType2 snaps the vectors to pixels, helping you eliminate legibility issues in dense UI text.
3. Leverage Composite Fonts for Localization

The FreeType2 module allows the engine to swap font files dynamically based on the character range.

Action: Use a Composite Font asset to define “Typeface” entries for different character sets (e.g., use one TTF for Latin and another for Kanji). This allows the engine to switch between different FreeType-processed files automatically, eliminating the “square box” (tofu) characters in localized builds.
4. Monitor the Font Cache Memory

FreeType2 rasterizes glyphs into a texture atlas known as the Font Cache.

Tip: Use the console command Slate.FlushFontCache if you notice UI memory growing too large after switching languages. This clears the textures generated by FreeType2 and recreates them only for the characters currently on screen, helping you eliminate unused glyph data from VRAM.
5. Prefer SDF/MSDF for High-Scale Text

In UE 5.5+, you can enable Signed Distance Field (SDF) or Multi-channel Signed Distance Field (MSDF) rasterization.

Best Practice: For large headings or text that scales dynamically, enable Distance Field Rendering in the Font Face asset. This allows FreeType2 to generate a resolution-independent representation, eliminating the blurry artifacts seen when standard rasterized text is scaled up.
6. Optimize Font Asset Sizes

Each unique combination of font size, bold/italic style, and outline settings requires FreeType2 to generate new glyphs.

Action: Minimize the number of different font sizes used in your UMG widgets. Standardizing on a few sizes allows the engine to reuse existing glyphs in the atlas, eliminating redundant rasterization calls and saving texture space.
7. Use Bounding Box for “Broken” Metrics

Some custom or decorative fonts have poorly authored internal metrics that cause clipping in UMG.

Tip: In the Font Face settings, change the Layout Method from “Metrics” to “Bounding Box.” This forces FreeType2 to calculate the line height based on the actual visual bounds of the characters, eliminating issues where the tops or bottoms of letters are cut off.
8. Validate Font Formats

FreeType2 is robust but can fail on highly complex or “corrupted” font files found on free font sites.

Action: If an imported font causes a crash or fails to render, run it through a font editor (like FontForge) to simplify the path data. Reducing the number of control points in the vector outlines helps eliminate processing bottlenecks and potential crashes within the FreeType2 parsing logic.
