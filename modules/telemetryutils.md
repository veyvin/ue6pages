### AI Description & Usage Tips

tility library designed to simplify the capture and reporting of development-cycle metrics. It provides the standardized API used by the Studio Telemetry system to track “iteration performance,” such as how long it takes to open the editor, compile shaders, cook assets, or package a build.

While standard analytics focus on player behavior, TelemetryUtils focuses on developer efficiency. It allows studio leads and pipeline engineers to eliminate guesswork regarding pipeline bottlenecks by collecting raw timing and success data from the local workstation and sending it to an aggregation backend like Horde Analytics.

Practical Usage Tips and Best Practices
Enable the Studio Telemetry Plugin
TelemetryUtils is most effective when the Studio Telemetry plugin is active. Go to Edit > Plugins to ensure it is enabled. This ensures that the underlying utilities are ready to log events, helping you eliminate blind spots in your build and iteration pipeline.
Configure Horde as the Backend
To make telemetry data actionable, configure your DefaultEngine.ini to point to a Horde server. Providing an APIServerET address allows the module to push metrics automatically. This helps you eliminate the need for manual spreadsheets or surveys to determine why local build times are regressing across the team.
Capture Custom Workflow Metrics
Use the TelemetryUtils API to wrap custom team-specific tools, such as an automated world-building script or a custom asset validator. Tracking how long these tools take to run across the whole team helps you eliminate inefficient workflows that are silently draining developer hours.
Monitor Zen DDC and Cook Efficiency
This module is deeply integrated with the Zen Loader and DDC (Derived Data Cache). Use the default telemetry events to monitor “Cache Hit” vs. “Cache Miss” ratios. High miss ratios indicate a DDC configuration issue that you must eliminate to restore fast asset loading speeds.
Analyze UBT and UBA Compilation Times
TelemetryUtils automatically tracks Unreal Build Tool (UBT) performance. By reviewing this data in a dashboard, you can identify if specific C++ modules are consistently slow to compile. This data helps you eliminate compilation bottlenecks by highlighting modules that need to be split or optimized.
Set Warning Thresholds
Use the aggregated data from this module to establish “performance baselines” for common tasks (e.g., “Editor startup should take less than 60 seconds”). If the telemetry shows a sudden spike across the team, it helps you eliminate the cause—such as a bad plugin or a corrupt network drive—before it impacts a milestone.
Isolate Local vs. CI/CD Performance
TelemetryUtils can report whether a process was run on a developer’s machine or a build farm. Comparing these metrics helps you eliminate discrepancies between local and remote environments, ensuring that your automation agents are performing as expected.
Manage Data Privacy and Elimination
Telemetry data generated by this module is private to your studio and is not sent to Epic Games. When a project concludes, you can manage the “elimination” of old telemetry indices in your Horde or database backend to eliminate storage bloat while keeping your active metrics fast and searchable.
