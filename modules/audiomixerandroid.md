Description and Purpose

This module serves as the hardware abstraction layer between the Unreal Audio Engine and the Android audio subsystem. Its primary role is to manage the output stream, handle hardware callbacks, and ensure that the digital audio signal generated by Unreal is correctly converted into a format the Android device can play. In modern versions of UE (5.x), this module primarily utilizes AAudio (high-performance, low-latency) or OpenSL ES (legacy compatibility) to interface with the device’s audio hardware. It is critical for ensuring feature parity on mobile, such as real-time submixing, DSP effects, and spatialization.

Practical Usage Tips and Best Practices
Prefer AAudio for Low Latency
In Project Settings > Platforms > Android, ensure that your target SDK levels allow for AAudio. This module performs best on devices running Android 8.1 (Oreo) and higher. AAudio is designed to eliminate unnecessary buffering steps, providing the lowest possible latency for gameplay sounds.
Adjust Callback Buffer Size
To balance performance and stability, tune the Callback Buffer Frame Size in the Android-specific Audio settings. Smaller buffers (e.g., 512 or 1024) result in lower latency but increase CPU overhead. If you hear “crackling” or “popping,” increase the buffer size to give the Android hardware more time to process the audio.
Configure Number of Buffers to Queue
Android devices vary wildly in hardware quality. Use the Number of Buffers to Queue setting (typically set to 2 or 3) to create a safety margin. A higher number of buffers will make the audio stream more resilient to OS-level hitches at the cost of slight input-to-sound latency.
Optimize Voice Count for Mobile CPU
Mobile CPUs are easily throttled. Use the settings provided by this module to limit the Max Channels (voices). While PC might handle 64 or 128 voices, a limit of 32 voices on Android is often sufficient to maintain high-quality sound while saving significant battery and CPU cycles.
Handle Audio Focus Changes
The AudioMixerAndroid module interacts with the Android OS to handle “Audio Focus.” Ensure your game logic responds correctly when a user receives a phone call or a notification. Unreal typically handles the muting/pausing automatically, but you should test that your background music resumes smoothly.
Verify Sample Rate Matching
Most Android devices operate natively at 48,000 Hz. Ensure your project’s sample rate matches the device’s native rate to avoid software resampling. Resampling in the AudioMixerAndroid module adds extra CPU load and can slightly degrade the audio quality of sharp transients.
Debug with LogAudioMixer
If you encounter issues specific to Android audio, use adb logcat and filter for LogAudioMixer. This will show you which backend (AAudio or OpenSL ES) the module initialized, the hardware sample rate, and if any “underruns” (buffer starvation) are occurring.
Audio Stability During Intense Events
During high-action moments, such as a multi-player elimination event with many overlapping explosions and VOIP, the CPU load spikes. Use Sound Concurrency settings to ensure that less important sounds are culled, allowing the AudioMixerAndroid module to prioritize the most critical audio data and prevent audible stuttering.
