Practical Usage Tips and Best Practices
Implement PSO Precaching to Prevent Hitching
DirectX 12 requires Pipeline State Objects (PSOs) to be compiled before use. Use the engine’s PSO Precaching system (introduced in 5.1 and refined in 5.6) to eliminate “shader stutter” by compiling necessary states during loading screens or in the background before the player encounters them.
Utilize DRED for GPU Crash Debugging
When the GPU hangs, use the Device Removed Extended Data (DRED) features by passing -dred in the command line. This allows the DX12 module to provide a breadcrumb trail and page fault information, helping you eliminate the guesswork involved in identifying which draw call or shader caused the crash.
Monitor VRAM with DX12-Specific Stats
Use the console command stat D3D12RHI to see detailed information about video memory allocation, including transient resources and descriptor heaps. This data is vital to eliminate memory leaks and ensure your project stays within the hardware limits of your target GPUs.
Leverage Agility SDK for Modern Features
Ensure your project is configured to use a recent version of the DirectX 12 Agility SDK in your Target.cs or project settings. This helps eliminate compatibility issues with newer hardware features like Shader Model 6.6+ and Atomic operations required for advanced Nanite workflows.
Configure Async Compute for Performance
Many rendering passes, such as screen-space reflections and certain lighting calculations, can run on the Async Compute queue. Ensure r.AsyncCompute is enabled (1) in your device profiles to eliminate idle GPU cycles and improve overall frame throughput by overlapping compute tasks with graphics tasks.
Manage Descriptor Heap Limits
DX12 has a finite number of descriptors it can track. If your scene has thousands of unique textures, you may hit heap limits. Use Virtual Texturing or texture atlasing to eliminate excessive descriptor switches and stay within the DX12 module’s resource constraints.
Enable GPU Breadcrumbs for Narrowing Down Crashes
In addition to DRED, use -gpubreadcrumbs to write marker data to the GPU timeline. If the engine crashes, the logs will show the last completed “breadcrumb,” helping you eliminate stable passes from your investigation and focus on the problematic rendering stage.
Validate Resource States via RHI Validation
During development, run the engine with -rhivalidation. This forces the DX12 module to perform strict checks on resource transitions (e.g., ensuring a texture is in the “Pixel Shader Resource” state before it is sampled). This practice helps eliminate subtle visual glitches and flickering caused by incorrect resource state management.
