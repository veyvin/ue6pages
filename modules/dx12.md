Practical Usage Tips and Best Practices
1. Implement PSO Precaching (UE 5.3+)

Shader compilation stutters are a common issue in DX12. Modern UE5 versions use PSO (Pipeline State Object) Precaching to eliminate these hitches.

Best Practice: Ensure r.PSOPrecache.Mode=1 is enabled in your DefaultEngine.ini. Additionally, use the FShaderPipelineCache::NumPrecompilesRemaining() function to display a “Compiling Shaders” progress bar on your loading screen. This ensures gameplay begins only after critical PSOs are ready, eliminating frame drops during initial asset loading.
2. Monitor VRAM with ‘stat D3D12RHI’

DX12 gives the engine more control over memory, but this can lead to “Out of Video Memory” crashes if not monitored.

Tip: Use the console command stat D3D12RHI to see a detailed breakdown of memory allocation. Look for “Local Video Memory Used” to see exactly how much VRAM the DX12 module is claiming. This helps you eliminate memory leaks or over-budget textures before they cause system-wide instability.
3. Enable SM6 for Advanced Features

To utilize the latest DX12 optimizations, your project should target Shader Model 6 (SM6).

Action: In Project Settings > Platforms > Windows, set the Targeted Redering Level to DirectX 12 (SM6). This enables features like Nanite Programmable Rasterization, which allows Nanite to handle masked materials and world position offset more efficiently, eliminating performance bottlenecks on complex foliage.
4. Use Async Compute for Post-Processing

DX12 supports Async Compute, which allows the GPU to process compute tasks (like SSAO or certain Niagara effects) in parallel with the main graphics pass.

Tip: Ensure r.AsyncCompute=1 is enabled. By utilizing idle GPU cycles during the depth-only or base-pass phases, you can eliminate total frame time overhead for heavy post-processing effects.
5. Debug with Agility SDK

UE5 uses the DirectX 12 Agility SDK to provide features that aren’t yet standard in the base Windows OS.

Best Practice: If users report that the game won’t launch in DX12 mode, verify they are on a compatible version of Windows 10⁄11. Shipping the Agility SDK DLLs with your packaged build (which Unreal does by default) helps eliminate “Entry Point Not Found” errors on older Windows installations.
6. Leverage Virtual Shadow Maps (VSM)

VSM is a DX12-only high-resolution shadowing method that replaces traditional Shadow Map Cascades.

Action: Enable Virtual Shadow Maps in Project Settings. VSM uses the DX12 module’s ability to handle sparse memory textures, allowing for cinematic-quality shadows while eliminating the need for complex shadow distance tweaking and “peter-panning” artifacts.
7. Optimize with Pipeline State Keep-In-Memory

To save system memory, the engine often deletes PSOs after they are compiled, which can cause a small hitch if the driver has to pull them from the disk cache later.

Tip: For high-end PC titles, use the console variable D3D12.PSOPrecache.KeepLowLevel=1. This keeps the low-level DX12 PSOs in system RAM. While it increases memory footprint, it can eliminate micro-stutters in fast-paced games where the driver cache is slow to respond.
8. Verify Nanite Compatibility

Nanite requires specific DX12 features like 64-bit atomics and Wave Intrinsics.

Action: If Nanite is not working on a specific hardware set, check the logs for “Nanite is not supported.” Use the -d3ddebug command-line argument to run the DX12 debug layer, which provides verbose output to help you eliminate hardware-level incompatibilities during the development phase.
