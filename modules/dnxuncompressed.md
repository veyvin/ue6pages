### AI Description & Usage Tips

nreal Engine that provides support for the Avid DNxUncompressed codec. It is primarily used within the professional video and broadcast pipelines to handle high-fidelity, uncompressed video data.

What it is and What it’s used for

Located as part of the Avid DNxHR plugin suite, this module enables Unreal Engine to decode and encode Avid’s “Uncompressed” flavor of the DNx standard. Unlike lossy codecs (like H.264 or standard DNxHR), DNxUncompressed is a bit-accurate, “no-excuses” format designed for the highest possible image quality in post-production.

Primary uses include:

Movie Render Queue (MRQ) Export: Exporting cinematics from Sequencer into a format that retains every bit of visual data for color grading in DaVinci Resolve or Avid Media Composer.
High-End Media Playback: Playing back uncompressed master files within a level (e.g., for virtual production backgrounds or LED volumes).
Video IO & Virtual Production: Handling high-bandwidth video input/output through professional capture cards (like AJA or Blackmagic) while maintaining a lossless signal.
Practical Usage Tips and Best Practices
1. Enable the Avid DNxHR Plugin

The DNxUncompressed module is not enabled by default. You must navigate to Edit > Plugins, search for Avid DNxHR/DNxMXF Media Plugin, and enable it. Once active and the editor is restarted, you will see the DNx options available in the Movie Render Queue settings.

2. Use for Final Mastering, Not Previews

DNxUncompressed generates massive file sizes (several gigabytes per minute of footage). Use this format only for final delivery or when moving to a color-grading suite. For daily “dailies” or internal reviews, use a compressed flavor like DNxHR SQ or ProRes 422 to save disk space.

3. Match Bit Depth to Output

The DNxUncompressed format is often used for 10-bit or 12-bit workflows. When exporting from MRQ, ensure your Post Processing and Render Settings are configured to output a high bit-depth signal. Exporting an 8-bit buffer into a DNxUncompressed container results in wasted disk space with no actual gain in fidelity.

4. Optimize Disk Throughput

Because of the immense data rate required for uncompressed video, you must host your project or export directory on an NVMe SSD or a high-speed RAID array. Standard HDDs will bottleneck, causing the elimination of smooth playback in the Media Player and significantly slowing down the export process.

5. Pair with WAV Audio Export

The DNx container used by this module typically focuses on the video stream. For the most reliable results in post-production, add a .wav Audio setting to your Movie Render Queue configuration. This ensures that a separate, high-quality audio file is generated alongside your uncompressed video.

6. Coordinate with Color Grading (ACES)

If you are using an ACES (Academy Color Encoding System) workflow, DNxUncompressed is the ideal container. It provides the necessary headroom for the wide color gamut and high dynamic range required by professional colorists, ensuring no data is clipped during the transfer from Unreal to the grading software.

7. Verify MXF Container Compatibility

Unreal Engine typically wraps this data in an .mxf container. Before a major delivery, verify that your intended video editing software (NLE) supports the specific DNxUncompressed MXF flavor generated by Unreal. Most modern versions of Premiere Pro, Avid Media Composer, and Resolve support this natively.

8. Strategic Elimination of Alpha Issues

If you require transparency (Alpha channel) in your export, verify the specific sub-format within the DNx settings. Not all flavors of DNx support alpha. If alpha is required for compositing, ensure you choose a “4:4:4” profile that explicitly supports an alpha channel to avoid solid black backgrounds in your final render.
