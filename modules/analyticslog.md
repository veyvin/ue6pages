### AI Description & Usage Tips

Unreal Engine. Its primary purpose is to intercept analytics events generated by your game and write them to a local file in JSON format instead of (or in addition to) sending them to a remote cloud provider.

This module is an essential debugging tool, allowing developers to verify that events—such as player progression, economy changes, or “elimination” statistics—are being recorded with the correct attributes before they are transmitted to a live analytics dashboard.

Practical Usage Tips and Best Practices
Configure via DefaultEngine.ini To enable local file logging, you must register the provider in your project configuration. This tells the engine to use the AnalyticsLog module to handle event data.
ini
	[Analytics]

	ProviderModuleName=FileLogging
Copy code
Use the Multicast Provider for Simultaneous Logging In development builds, use the AnalyticsMulticast provider to send data to both your live dashboard (e.g., Flurry or Epic) and the local log file simultaneously. This allows you to cross-reference local data against what appears on your web console.
ini
	[Analytics]

	ProviderModuleName=AnalyticsMulticast

	ProviderModuleNames=FileLogging,YourRemoteProvider
Copy code
Locate Output Files for Verification The module generates files with a .analytics extension located in your project’s Saved/Analytics/ folder. These files contain raw JSON strings of every event, including timestamps and attribute maps, making them easy to parse with external scripts.
Audit “Elimination” and Gameplay Events Use the local log to audit complex event strings. If you are tracking an “elimination” event, you can verify in the JSON that attributes like WeaponUsed, Distance, and PlayerCoordinates are being passed as the correct data types (String vs. Number).
Strip from Production Builds Local logging is performance-heavy and creates ever-growing files on the user’s disk. Always wrap your provider configuration in a [AnalyticsDevelopment] or [AnalyticsTest] section to ensure it is “eliminated” from the shipping build.
ini
	[AnalyticsDevelopment]

	ProviderModuleName=FileLogging
Copy code
Integrate with C++ for Custom Logic If you need to trigger logic when a log is written, include the module in your Build.cs. Note that the provider is accessed via the generic IAnalyticsProvider interface.
C++
	// In YourProject.Build.cs

	if (Target.Configuration != UnrealTargetConfiguration.Shipping)

	{

	    PrivateDependencyModuleNames.Add("AnalyticsLog");

	}
Copy code
Check for Warnings in the Output Log If analytics events are not appearing in your .analytics file, check the standard Unreal Output Log. The module will print a warning if a session was not started correctly using StartSession(), which is a prerequisite for any data being recorded.
Use for Offline Testing The AnalyticsLog module is ideal for testing analytics in environments without internet access or behind strict firewalls, ensuring your instrumentation logic is robust regardless of network connectivity.
